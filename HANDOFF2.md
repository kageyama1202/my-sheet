# 案件管理システム 引き継ぎメモ（2026-04-01 最新版・メーラー実装済）

## 使い方
新しいClaude等のAIセッションで以下のように言ってください:
```text
https://raw.githubusercontent.com/kageyama1202/my-sheet/main/HANDOFF2.md を読んで。
XXXを修正したい。
修正対象ファイルも同様に:
action.htmlを読んで
→ https://raw.githubusercontent.com/kageyama1202/my-sheet/main/action.html
コード全文を貼り付ける必要はありません。

--------------------------------------------------------------------------------
プロジェクト概要
建設案件の管理システム。GitHub Pages + Firebase Realtime Database で構成。 GASは使わない方針。UIは全てスタンドアロンHTML。
リポジトリ: https://github.com/kageyama1202/my-sheet (public)
公開URL: https://kageyama1202.github.io/my-sheet/
ファイル構成（9ファイル + 2共通）
index.html — トップページ。Firebase→LocalStorage保存。通し番号(seqNo)自動付与。
today.html — フィルタ付き案件リスト。期間指定、ステータス/区分/会社/地域チェック。共通モーダル使用。下見予定日列あり。
calendar.html — 月表示カレンダー。施工/下見/下見予定の3種チェック切替。
map.html — Leaflet.js地図。Nominatimジオコーディング+Firebaseキャッシュ。ステータス色分け。
action.html — アクションダッシュボード。2タブ構成（アクション/備忘録）。5ステータスサブタブ。ランク制ステータス。共通モーダル使用。ソート切替（デフォルト/施工日順）。
comm.html — 通信タイムライン。2カラム。手動案件紐付け。未分類メッセージ赤ボーダー訴求。
admin.html — 管理者用CSVアップロード。SFのcsvData（手配ID）をキーとして保存。
mailer-test.html — ✉️ 【新規追加】一括送信パネル型メーラー。SMS/メールアプリのワンクリック起動と履歴保存機能。（※テスト完了後、本番組み込み予定）
shared-modal.css / shared-modal.js — 共通モーダルロジック。
Firebase構造 (主要部分)
app_tasks/[手配ID]/
  csvData: [配列] — CSVデータ（index 0〜12）
  mailStatus: { reform: true, takara: true, sms: true } — メーラーからの送信履歴（緑バッジ用）

app_parked_mails/[自動キー]/ 
  status: "parked" — 未送信/履歴保存用の仮置きメールデータ
  sendMethod: "sms" または "email"
  mailType: "reform", "takara", "sms"
CSVデータのカラムマッピング（実データで確認済み）
index 0: 手配: ID ★Firebaseキーの元
index 1: 手配ステータス（依頼/下見実施中/下見日程確定/下見完了/完了）
index 2: 施工日(数式用)
index 3: 工事区分名（略称）（新/ﾘ）
index 4: 手配: 管理名（案件名として表示に使用）
index 5: 打合担当者（会社名+担当者。スペース区切り1語目=会社名）
index 6: 担当者携帯許容記号除外（電話リンク対象）
index 7: 下見依頼日
index 8: 工務担当者(名称) ★タカラ側担当者名
index 9: 施工業者
index 10: 施工指示および備考
index 11: 住所(改行なし)
index 12: 下見日
新規実装：メーラー機能（mailer-test.html）の仕様
一括表示パネル: 案件を選ぶと「リフォーム問合」「施工日確定(タカラ)」「施工日確定(先方・業者)」の3ブロックが縦に並んで表示される。
工事区分による表示切替: CSVの工事区分(index 3)に「ﾘ」が含まれない場合、リフォーム問合ブロックは自動で非表示になる。
強力な連絡先検索: app_contacts（連絡帳）を「フルネーム」→「会社名」→「担当者名」の順で走査し、見つかったメールアドレスと電話番号を自動セットする（土屋ホーム等の会社メアドも取得可能）。
ワンクリック起動: 画面上のボタンから、端末の標準SMSアプリまたはメールソフトを直接起動できる。起動前にテキストエリアで文章の加筆修正も可能。
履歴保存とアラート: アプリ起動後「履歴保存」を押すとバッジが「🟢済」になる。保存していない（仮置き状態の）メッセージがある場合は、画面上部に赤いアラートで一覧表示される。
完了案件の非表示: ステータス(index 1)が「完了」の案件はリストに表示されない。
既知の問題・未完了タスク
高優先度
comm.htmlの自動振り分け（電話番号マッチ+案件1件だけの場合は自動assignedCase）
今回作成した mailer-test.html の機能を、本番の shared-modal.js へ統合するか検討。
中優先度
マップのジオコーディング完走確認（124件中75件程度取得済み）
施工条件メモの構造化（app_contractor_notesの表示UI）
連絡先のFirebase取り込み改善（Mac連絡先の名前クリーニング精度）
shared.js共通化の続き（モーダル以外のユーティリティ関数）
低優先度
AI下見スケジュール提案ページ
担当者マスターシート
キーワードタブのクロスデバイス共有（現在localStorage）
Google Drive API連携（ファイル自動アップロード）
注意事項・開発ルール
Kの作業スタイル: 完全ファイル置換を好む。部分修正より全置換が確実。
var使用推奨: const/letの初期化順序バグが何度も発生。varが安全。配列インデックス指定時はシステム誤認を防ぐため csv[ 2 ] のようにスペースを入れること。
GitHubキャッシュ: Ctrl+Shift+Rで強制リロード必須。
リポジトリはpublicなので、APIキー・電話番号・名前等の個人情報をコードに含めない。
カラム定義・スプレッドシート構造・業務フローは聞くな、このメモを参照しろ。
